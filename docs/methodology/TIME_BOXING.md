# 📦 The Time Boxing Method

**Goal:** Create a "Synthetic Sense of Urgency" to defeat Parkinson's Law.
*Reference: [The Time Boxing Method](https://super-productivity.com/guides/time-boxing-method/)*

> **Parkinson's Law:** "Work expands to fill the time available for its completion."

## 1. Boxing vs. Blocking

*   **Time Blocking:** "I will do *coding* from 9am to 11am." (Category focused)
*   **Time Boxing:** "I will *finish the API endpoint* by 10:30am." (Output focused)

**We use Time Boxing.** It forces you to scope the work to fit the box.

## 2. The Workflow

1.  **Brain Dump:** List everything you *could* do today.
2.  **Select 3 Outcomes:** Pick the 3 items that actually move the needle.
3.  **Tetris (Estimation):** Assign time estimates to them.
    *   *Be Pessimistic:* If you think it takes 1 hour, box it for 1.5 hours.
4.  **Execute:** Start the timer. The deadline is real.

## 3. Managing "The Overflow"

What if you don't finish in the box?

1.  **Stop:** Acknowledge the box is closed.
2.  **Assess:** Is it worth opening a *new* box? Or is it good enough for now?
3.  **Re-Schedule:** If it must be done, move it to the next available slot. Do not just "push through" and ruin the rest of the day's schedule.

## 4. Super Productivity Usage
*   Use the **Schedule** view (`S` key) to see your boxes.
*   Use the **Daily Plan** feature to force this process every morning.
