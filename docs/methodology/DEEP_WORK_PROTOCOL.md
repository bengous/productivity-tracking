# 🛡️ Deep Work Protocol for Developers

**Goal:** Protect your "Flow State" from the modern attention economy.
*Reference: [Deep Work for Developers](https://super-productivity.com/guides/deep-work-for-developers/)*

## 1. The Cost of Interruption
*   **Attention Residue:** It takes **20+ minutes** to regain full context after a distraction.
*   **Context Switching:** Every time you check Slack while coding, you delete your mental RAM.

## 2. The Protocol

### 🔒 Phase 1: Fortress Mode (Setup)
Before you write a line of code:
1.  **Notifications OFF:** Phone DND, Slack paused.
2.  **Visual Signal:** Headphones ON.
3.  **Select ONE Task:** In Super Productivity, select *only* the task you are doing.

### ⚔️ Phase 2: The Loop (Execution)
1.  **Time Box:** Set a timer for **60-90 minutes**.
2.  **Capture:** If a thought pops up ("I need to email X"), use `Shift+A` to quick-add it to the backlog, then **immediately return to code**. Do not process it.
3.  **Refuse:** If someone interrupts, say: "I'm in a deep block. Can we sync at [Next Hour]?"

### 🛑 Phase 3: Decompression
1.  **Stop:** When the timer hits 0, STOP.
2.  **Detach:** Step away from the screen. Walk, stretch, or get water.
3.  **Review:** Look at what you accomplished. Dopamine hit!

## 3. Scheduling Strategies

*   **Rhythmic:** Same time every day (e.g., 08:00 - 11:00 is ALWAYS code).
*   **Bimodal:** Entire days dedicated to deep work (e.g., "No-Meeting Wednesdays").
